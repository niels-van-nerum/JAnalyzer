The application contains a three threaded design, a capture thread, an analysis thread and a transmitting thread. These will work on separate threads. We need a way to safely deal with these concurrent processes. That is why we choose to use a LinkedBlockingQueue<byte[]>. The byte array is chosen because that is the output of our audio sample. We need to make sure that we do not keep putting identical references to the byte array in the queue. In order to do this we use Arrays.copyOf to create a fresh byte array instance in memory. We will use a bounded queue to prevent the application from growing infinitly if the producer is quicker then the consumer. This prevents ever-growing memory. Lastly we use offer() to add things to the queue, and drop it if the queue is full. This way, we can continue consuming from the audio buffer, and we make sure that we are always keeping up consuming fresh audio data.
